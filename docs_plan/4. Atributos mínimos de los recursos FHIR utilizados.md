# Atributos mínimos de los recursos FHIR utilizados

---

## 1. Organization
Se usará para:
- ACME Salud
- Aseguradoras

**Campos mínimos**
- id
- name
- identifier
- type (prestador / aseguradora)

---

## 2. Location
Representa las sedes físicas (Norte, Centro, Sur).

**Campos mínimos**
- id
- name
- identifier
- managingOrganization (referencia a Organization)

---

## 3. HealthcareService
Representa cada servicio médico.

**Campos mínimos**
- id
- name
- providedBy (Organization)
- location (Location)

---

## 4. Practitioner
Representa al médico especialista.

**Campos mínimos**
- id
- identifier (tarjeta profesional)
- name

---

## 5. PractitionerRole
Relaciona médico con institución, sede y servicio.

**Campos mínimos**
- id
- practitioner (referencia a Practitioner)
- organization (Organization)
- location (Location)
- healthcareService (HealthcareService)

---

## 6. Patient
Representa al paciente.

**Campos mínimos**
- id
- identifier
- name

---

## 7. Coverage
Relaciona paciente con aseguradora.

**Campos mínimos**
- id
- beneficiary (Patient)
- payor (Organization – aseguradora)
- status

---

## 8. Schedule
Representa la agenda mensual.

Puede pertenecer a:
- HealthcareService
- PractitionerRole

**Campos mínimos**
- id
- actor (HealthcareService o PractitionerRole)
- planningHorizon (inicio y fin del mes)

---

## 9. Slot
Representa cada intervalo de tiempo disponible.

**Campos mínimos**
- id
- schedule (Schedule)
- start
- end
- status (free / busy)

---

## 10. Appointment
Representa la cita médica.

**Campos mínimos**
- id
- status
- start
- end
- participant
  - Patient
  - PractitionerRole
- serviceType (tipo de consulta)
- slot (referencia al Slot reservado)

---

## 11. AppointmentResponse
Confirmación al paciente.

**Campos mínimos**
- id
- appointment
- actor (Patient)
- participantStatus (accepted)
